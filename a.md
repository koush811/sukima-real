# Sukima Real

## 概要
- このアプリは、1日のスケジュール表から隙間時間を検出し、その間にユーザーに写真を撮影させ、隙間時間を共有することができるWEBアプリです。

## 環境
- フロントエンド: React(js)
- バックエンド: Firebase
    - Authentication: Googleログイン
    - Firestore: データベース(投稿内容を保存)
    - Storage: 写真を保存
    - Hosting: Web公開

## 画面構成
- 画面はスマホ、PCどちらにも対応させる
### ログイン画面
    - Googleでログイン
### ホーム画面
    - パネル(下部)
        1. PHOTO: 押すとカメラが起動→写真を撮影→撮影した写真を投稿
        2. SCHEDULE: 押すとスケジュール画面に遷移
        3. POSTS: 押すと投稿一覧画面(ホーム画面)に遷移
    - 投稿一覧
        - 投稿された写真を表示
        - 写真をクリックすると、投稿詳細画面に遷移
            - 詳細画面では、投稿された写真の撮影日時、撮影者を表示
            - また下部にリアクションボタンを配置(👍,😊,💖)
###　スケジュール画面
    - GoogleカレンダーのAPIを使用して、ユーザーのスケジュールを取得し、表示
### その他
    - ページ上部中央にアプリ名(Sukima Real)と現在時間00:00を表示
    - 右上にアイコンを表示する(押すとウィンドウを表示)
        - アイコンは円形

## 機能要件

### ログイン機能
- Googleアカウントでログインする
- 初回ログイン時はFirestoreにユーザー情報を作成する
初回ログイン
↓
Firestore作成
↓
ホーム

- 2回目以降は既存ユーザーとしてログインする
- 認証に成功すれば、ホーム画面に遷移し、失敗すれば「ログインに失敗しました」を表示する

### スケジュール表示
- パネルのSCHEDULEボタンを押すと、スケジュール画面に遷移する
- GoogleカレンダーAPIを使用して、ユーザーのスケジュールを取得し、表示する
- Google Calendarの閲覧権限(calendar.readonly)を取得する
- 取得するのは当日のスケジュールのみ
- カレンダー取得に失敗した場合は
「予定を取得できませんでした」
を表示する
- 予定が存在しない場合は1日に3回ランダムな時間に３０分の隙間時間を設ける

### すきま時間検出機能
- GoogleカレンダーAPIから当日の予定を取得する
- 予定と予定の間の空き時間を算出する
- 空き時間が30分未満の場合はその間すべてをすきま時間とする
- 空き時間が30分以上の場合は最初の30分間をすきま時間とする
- 1日に複数のすきま時間が存在する場合はランダムに1つ選択する
- 現在時刻がすきま時間内の場合のみPHOTOボタンから投稿できる

### 投稿機能
- すきま時間内のみ使用可能
- 隙間時間外の場合は、PHOTOボタンを押すと「現在はすきま時間ではありません
次のすきま時間15:00〜15:30」と表示する
- パネルのPHOTOボタンを押すと、カメラが起動し、写真を撮影できる
- 撮影した写真は、Firebase Storageに保存され、Firestoreに投稿内容(撮影日時、撮影者、写真のURL)を保存する
- 投稿一覧には、Firestoreに保存された投稿内容を表示する
- 投稿の回数は1つの隙間時間に1回までとする
- 1回目以降はPHOTOボタンを押せないようにする

↓

↓
判定完了
↓

### 投稿一覧表示機能(Home画面)
- パネルのPOSTSボタンを押すと、投稿一覧画面(ホーム画面)に遷移する
- 投稿一覧には、Firestoreに保存された画像を表示する
- 投稿一覧は、撮影日時の新しい順に表示する
- スマホ、PC関係なく横3列表示とする


### Sukima度判定機能
- 投稿された写真をgeminiAPIに送信し、Sukima度を判定する
- Sukima度とは、写真の中でどれだけSukimaを表現できたかを計る指標である
- Sukima度は、0~100の数値で判定され、判定結果は投稿詳細画面に表示する

### 詳細表示機能
- 投稿一覧の写真をクリックすると、詳細画面に遷移する
- 画像を大きく表示する
- 詳細画面では、投稿された写真の撮影日時、撮影者、Sukima度を表示する
- また下部にあるリアクションボタンを押すと、リアクション数が増えるようにする
- 投稿詳細画面のリアクション数は、Firestoreに保存されるようにする
- リアクションは自分の投稿には押せないようにする
- リアクションは、1回しか押せないようにする

### ユーザーパネル
- 右上のアイコンを押すと、ユーザーパネルが表示される
- ユーザーパネルには、ユーザーのプロフィール情報(ユーザーネーム、メールアドレス)を表示する
- ユーザーパネルには、ログアウトボタンを配置し、押すとログアウトするようにする

### 投稿削除機能
- 自分の投稿の詳細画面に削除ボタンを配置し、押すと投稿を削除できるようにする
- 投稿削除時には、Firestoreから投稿内容を削除し、Firebase Storageから写真を削除する
- Reactionも削除する

## Sukima度

### 概念
- Sukima度とは、写真の中でどれだけSukimaを表現できたかを計る指標である
- Sukima度は、0~100の数値で判定される
### 判断基準
・"すきま"が映っているか(家具と家具の間や部屋の角、側溝など)
・リラックスしている様子
・日常らしさ
・隙間時間らしさ
・自然な雰囲気

これらを総合して
0〜100点で評価する

## API
- GoogleカレンダーAPI: ユーザーのスケジュールを取得するために使用する
- geminiAPI: 投稿された写真を送信し、Sukima度を判定するために使用する
    - プロンプト
        あなたはSukima度を判定するAIです。

        以下の基準で0〜100点で評価してください。

        ・写真内に物理的な隙間があるか
        ・隙間が写真の主題になっているか
        ・リラックスした雰囲気があるか
        ・日常らしい写真か

        JSON形式で返してください。
        
        例：
        {
        "score": 85,
        "reason": "家具の隙間が明確で、日常感も伝わります。"
        }
- APIkeyは、Firebase Functionsに保存し、フロントエンドからは直接アクセスできないようにする
